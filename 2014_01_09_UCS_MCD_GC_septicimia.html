I checked in with David Phillips about what he did on this topic, and determined that it is worthy of some additional investigation on my part.

From: David Phillips 
Sent: Wednesday, January 08, 2014 10:33 PM
To: Abraham D. Flaxman
Subject: RE: GC redistribution with MCD data

Hey Abie,

First of all, this project has been put on the back-burner, mostly because I think nobody wants to update the garbage code redistribution code again.

Anyway, what I did was make a ton of dummy variables and used them in many logistic regressions. The outcome was whether or not cause X was the underlying cause in a given death certificate, and the explanatory variables were dummy variables for each cause in each other field on the death certificate. So, there were about 500 dummies (all of the GBD causes of death plus all ICD10 garbage codes) multiplied by the 20-something fields. I also tinkered around with age and country fixed effects, but never got far enough to evaluate performance. A separate regression was run for each of the different possible outcome variables; each of the 235 GBD causes of death. I fit the regression amongst only observations that didnt have garbage as underlying, and then predicted amongst the garbage observations. The results were then stored such that each garbage observation was given a distinct probability of actually being each of the GBD causes. I then summed probabilities across all observations and redistributed garbage with respect to the proportions represented by the sums. I also tried just assigning each observation to its most likely cause.

The main problem I ran into was complete (or quasi-complete) separation, which, in hindsight, we should have seen coming given the abnormal number of explanatory variables and the fact that each variable was quite sparse (just a few 1s amongst millions of 0s). I aggregated garbage codes and tried again, but there were still way too many sparse variables to be workable. Kyles idea was to also aggregate GBD codes, and then we would do some kind of cascading thing, but I havent tried to tackle that yet.

Because thousands of dummy variables is a silly way to store a few categorical variables, I decided to run random forest. This worked fine, but the results were still quite contrary to Mohsen and Chris expectations, which is basically where we all lost hope. 

Thats the whole story! What are you working on?
David

For a qualitative validation, he made a plot of redistribution for septicimia garbage stacked by age group, which showed implausible results, such as most septicimia redistributed to RHD. I wonder what my redistribution plot looks like (it should be a table, not a plot, also).
